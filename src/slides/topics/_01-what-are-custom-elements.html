Defining a customElement

      
// standard way to define custom elements
class PixuButtonStd extends HTMLButtonElement {
  constructor() {
    super();    
    this.setAttribute('role', 'button');
  }
}

// register in the standard way
customElements.define(
  'pixu-button-std', PixuButtonStd
);

// usage
<pixu-button-std>
  Click me!
</pixu-button-std>

// runtime
<pixu-button-std type="button">
  Click me!
</pixu-button-std>
      
    
      
// extends pattern way to define custom elements
class PixuButton extends HTMLButtonElement {
  constructor() {
    super();    
    this.setAttribute('role', 'button');
  }
}
  
// register with the extends option
customElements.define(
  'pixu-button', PixuButton,
  { extends: 'button' }
);

// usage
<button is="pixu-button">
  Click me!
</button>

// runtime
<button is="pixu-button" type="button">
  Click me!
</button>

Can you see the difference?

  • Both are defined using class syntax
  • Both are registered with customElements.define
  • But the extends pattern keeps the native element's behavior (and a11y/SEO), differently than the standard pattern
customElements let us extend native HTML elements with custom behavior while maintaining all their built-in functionality, accessibility, and browser support.
